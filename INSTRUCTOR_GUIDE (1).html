⚙️ How the System Works

Role-Based Login

When a user logs in, the system checks their role (admin or customer). Admins are automatically redirected to /admin. Customers go to the shop. The AdminMiddleware blocks any non-admin from accessing admin routes — they receive a 403 error.

Admin Panel

The admin panel is a separate section of the site accessible only to admin users. From there, the admin can manage every part of the store: add/edit/delete categories, subcategories, products, view and update orders, and manage users including changing their role instantly from a dropdown.

Shop (Customer Side)

Customers can browse all products, filter by category, search by name, and sort by price. They can add items to their cart (stored in the session), go through checkout, and view their order history. No payment gateway is implemented — orders are placed directly.

Categories & Subcategories

Products belong to a category and optionally a subcategory. When adding a product, the subcategory dropdown filters automatically based on the selected category using JavaScript. This is managed through relational foreign keys in the database.
